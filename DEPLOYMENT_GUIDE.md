## 🚀 Deploy to Render.com

### Step 1: Sign in to Render.com

1. Go to [https://render.com](https://render.com)
2. Sign in with your GitHub account

### Step 2: Create New Web Service

1. Click **"New +"** button in the top right
2. Select **"Web Service"**
3. Connect your GitHub repository: `harsha895155/sampleproject`
4. Render will automatically detect your `render.yaml` file

### Step 3: Configure Environment Variables

You need to set these environment variables in Render dashboard:

#### Required Variables:

1. **MONGO_URI** or **MASTER_DB_URI**
   - Your MongoDB connection string
   - Example: `mongodb+srv://username:password@cluster.mongodb.net/auth_db`
   - Get this from MongoDB Atlas

2. **TENANT_DB_BASE_URI**
   - Base URI for tenant databases
   - Usually same as MONGO_URI without the database name
   - Example: `mongodb+srv://username:password@cluster.mongodb.net/`

3. **JWT_SECRET**
   - Will be auto-generated by Render
   - Or set your own secure random string

4. **VITE_API_URL**
   - Your Render app URL (you'll get this after deployment)
   - Example: `https://your-app-name.onrender.com`

5. **EMAIL_USER**
   - Your email address for sending notifications
   - Example: `your-email@gmail.com`

6. **EMAIL_PASS**
   - Your email app password (not regular password)
   - For Gmail: Generate an App Password from Google Account settings

7. **FAST2SMS_API_KEY** (Optional)
   - Your Fast2SMS API key for SMS notifications
   - Leave empty if not using SMS

### Step 4: Deploy

1. Click **"Create Web Service"**
2. Render will:
   - Install dependencies
   - Build the client
   - Start the server
3. Wait for deployment to complete (5-10 minutes)

### Step 5: Update VITE_API_URL

After first deployment:
1. Copy your Render app URL (e.g., `https://your-app-name.onrender.com`)
2. Go to Environment Variables in Render dashboard
3. Update **VITE_API_URL** with your app URL
4. Trigger a manual redeploy

## 📝 Important Notes

1. **Free Tier Limitations:**
   - Service spins down after 15 minutes of inactivity
   - First request after spin-down takes 30-60 seconds
   - 750 hours/month free

2. **MongoDB Atlas:**
   - Make sure your MongoDB Atlas cluster allows connections from anywhere (0.0.0.0/0)
   - Or add Render's IP addresses to whitelist

3. **Email Configuration:**
   - For Gmail, enable 2-factor authentication
   - Generate an App Password from: https://myaccount.google.com/apppasswords
   - Use the App Password, not your regular Gmail password

## 🔧 Troubleshooting

### Build Fails
- Check build logs in Render dashboard
- Verify all dependencies are in package.json
- Ensure Node version compatibility

### Database Connection Issues
- Verify MONGO_URI is correct
- Check MongoDB Atlas network access settings
- Ensure database user has proper permissions

### Email Not Sending
- Verify EMAIL_USER and EMAIL_PASS are correct
- For Gmail, use App Password, not regular password
- Check email service provider settings
